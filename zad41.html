Rodzaje pętli w programowaniu

Pętla for

Konstrukcja:

for(inicjalizacja; warunek; inkrementacja) { // blok kodu do wykonania }

Opis:

Pętla for jest używana, gdy znamy dokładną liczbę iteracji, które chcemy wykonać. Składa się z trzech części: inicjalizacji (ustawienie początkowych wartości), warunku (sprawdzanego przed każdą iteracją) oraz wyrażenia modyfikującego (zwykle inkrementacja lub dekrementacja licznika).

Schemat blokowy:

Inicjalizacja → Warunek → (Tak) → Blok kodu → Modyfikacja
↓ (Nie)

Pętla while (z kontrolowanym wejściem)

Konstrukcja:

while(warunek) { // blok kodu do wykonania }

Opis:

Pętla while wykonuje blok kodu tak długo, jak spełniony jest określony warunek. Warunek jest sprawdzany przed każdą iteracją (kontrolowane wejście). Jeśli warunek jest fałszywy już na początku, pętla nie wykona się ani razu.

Schemat blokowy:

Warunek → (Tak) → Blok kodu
↓ (Nie)

Pętla do while (z kontrolowanym wyjściem)

Konstrukcja:

do { // blok kodu do wykonania } while(warunek);

Opis:

Pętla do while podobna jest do pętli while, ale z kluczową różnicą - warunek jest sprawdzany po wykonaniu bloku kodu (kontrolowane wyjście). Oznacza to, że pętla zawsze wykona się przynajmniej raz, nawet jeśli warunek jest fałszywy już na początku.

Schemat blokowy:

Blok kodu → Warunek → (Tak) → Powrót
↓ (Nie)
